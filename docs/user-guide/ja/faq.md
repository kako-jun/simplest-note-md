# よくある質問（FAQ）

> Agasteerについてよくある質問と回答をまとめました。

---

## 基本的な質問

### Q: サーバーは必要ですか？

**A:** いいえ、完全にブラウザ上で動作します。静的ホスティング（Cloudflare Pages、GitHub Pages等）で公開できます。

### Q: オフラインで使えますか？

**A:** はい、一度読み込めばオフラインで編集できます。ただし、GitHub同期にはインターネット接続が必要です。

### Q: モバイルでも使えますか？

**A:** はい、レスポンシブデザインでスマホでも快適に編集できます。PWA対応なので、ホーム画面に追加すればネイティブアプリのように使えます。

### Q: GitHub連携は必須ですか？

**A:** いいえ、GitHub連携なしでも動作します。ただし、ブラウザのデータはキャッシュクリアで消える可能性があるため、大切なメモを残すならGitHub連携を推奨します。

## データ管理

### Q: データはどこに保存されますか？

**A:** 3箇所に保存されます。

| データ         | 保存先                        |
| -------------- | ----------------------------- |
| ノート・リーフ | IndexedDB（ブラウザ）+ GitHub |
| 設定情報       | LocalStorage（ブラウザ）      |
| フォント・背景 | IndexedDB（ブラウザのみ）     |

### Q: 既存のMarkdownファイルをインポートできますか？

**A:** はい、2つの方法があります。

1. **インポート機能**: 設定画面から Agasteer形式（ZIP）またはSimpleNote形式（JSON）をインポート
2. **手動配置**: GitHub上の `.agasteer/notes/` にMarkdownファイルを配置し、Pullを実行

### Q: データのバックアップは必要ですか？

**A:** 推奨します。IndexedDBはブラウザのキャッシュクリアで消える可能性があります。

- GitHub同期を設定してバックアップ
- ZIPエクスポートでローカルに保存

### Q: ノート階層を3階層以上にできますか？

**A:** いいえ、2階層（ノート → サブノート）までです。シンプルさを保つための設計です。

## 同期について

### Q: 複数のデバイスで同期できますか？

**A:** はい、同じGitHubリポジトリにPull/Pushすることで同期できます。

**注意**: 競合解決UIはまだ実装されていません。複数デバイスで同時編集すると、後からPushしたデータで上書きされます。

### Q: 通信量や費用はどれくらいですか？

**A:** GitHubは無料プランで十分です。

| 利用パターン | Pull/Push回数 | 費用     |
| ------------ | ------------- | -------- |
| 軽い利用     | 〜30回/月     | **無料** |
| 通常利用     | 〜100回/月    | **無料** |
| ヘビー利用   | 〜300回/月    | **無料** |

GitHub APIのレート制限（5,000回/時間）内であれば費用は発生しません。

## 画像・メディア

### Q: 画像は埋め込めますか？

**A:** Markdownの画像構文は使用できますが、画像アップロード機能はありません。

方法:

1. GitHub上の `images/` に画像を配置し、`![説明](./images/example.png)` で参照
2. 外部の画像ホスティングサービス（Imgur等）を使用

### Q: 動画や音声は埋め込めますか？

**A:** Markdown内でHTMLタグを使えば可能です。

```html
<video src="https://example.com/video.mp4" controls></video>
```

## トラブルシューティング

### Q: Pushが失敗します

以下を確認してください:

1. GitHub Tokenが正しいか
2. トークンに `repo` スコープ（または Contents: Read and write）が付与されているか
3. リポジトリ名が `owner/repo` 形式で正しいか
4. トークンの有効期限が切れていないか

### Q: Pullが失敗します

以下を確認してください:

1. GitHub Tokenが正しいか
2. リポジトリ名が正しいか
3. リポジトリが存在するか

### Q: データが消えました

考えられる原因:

1. ブラウザのキャッシュをクリアした
2. プライベートモードで使用していた
3. 別のブラウザ/プロファイルで開いている

**対策**: GitHub同期を使用している場合、Pullを実行すればデータを復元できます。

### Q: Vimモードが効かない

以下を確認してください:

1. 設定画面でVimモードがONになっているか
2. エディタをクリックしてフォーカスしているか
3. Escキーでノーマルモードに戻っているか

### Q: Androidで空行をタップするとスクロールがジャンプする

**A:** Gboard（Google日本語入力）との相性問題です。

空行（段落と段落の間の空白行）をタップすると、スクロール位置が意図しない場所にジャンプすることがあります。

**回避策**: 空行をタップする代わりに、テキストがある行をタップしてからカーソルを移動してください。

### Q: ヘッダー左端の縦線は何？

**A:** 次のStaleチェックまでの残り時間を示す進捗バーです。

- 5分間隔でリモートの変更を自動チェックします
- バーが伸びきると（5分経過）チェックが実行されます
- 他のデバイスで変更があった場合、Pullボタンに赤丸が表示されます
